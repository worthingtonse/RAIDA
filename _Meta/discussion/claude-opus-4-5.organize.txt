=============================================================================
DOCUMENTATION ORGANIZATION ANALYSIS - Claude Opus 4.5
=============================================================================

## Current State

The documentation has grown organically with some folders that are:
1. Core protocol specification (client-server wire format)
2. Implementation guidance (tutorials, best practices)
3. Supplementary material (security, requirements, data files)
4. Empty/legacy folders
5. Working files (model analysis .txt files)

This mix can confuse AI systems trying to understand what IS the protocol
versus what is guidance ABOUT the protocol.

=============================================================================
FOLDER CLASSIFICATION
=============================================================================

### CORE PROTOCOL (Wire Format Specification)
These define EXACTLY what bytes go on the wire between client and server.

| Folder | Content | Keep |
|--------|---------|------|
| 00_Introduction/ | Overview, glossary, architecture | YES |
| 01_Protocol_Fundamentals/ | Packet structure, byte order, padding | YES |
| 02_Request_Headers/ | 32-byte request header format | YES |
| 03_Response_Headers/ | Response header, status codes | YES |
| 04_Cryptography/ | AES-CTR encryption for protocol | YES |
| 05_Challenge_Response/ | CRC32, challenge format | YES |
| 06_Denominations/ | Token value encoding (used in protocol) | YES |
| 06_Terminator/ | 0x3E3E packet terminator | YES |
| 07_Commands/ | All RAIDA commands with byte formats | YES |
| 08_Networking/ | UDP/TCP transport, ports | YES |

"For RAIDA protocol wire format, ONLY use folders 00-08.

- Mixing RFP requirements with actual protocol specification

### Step 5: Add README files

Add to root README.md (or create STRUCTURE.md):
```markdown
## Documentation Scope

**Protocol Wire Format (folders 00-08):**
These define EXACTLY what bytes go on the network between client and server.
Use these exclusively for implementing RAIDA client/server communication.

**Supplementary Material (folder 80_Supplementary/):**
Implementation guides, best practices, file formats, and reference material.
NOT part of the protocol specification.

**Working Files (discussion/):**
Temporary analysis files from documentation development.
```

Add 80_Supplementary/README.md:
```markdown
# Supplementary Documentation

This folder contains material that is NOT part of the RAIDA protocol
wire format specification.

For protocol implementation (what bytes go on the network), see folders
00-08 in the parent directory.

## Contents
- Client_Guide/ - Tutorial for building RAIDA clients
- File_Formats/ - Token STORAGE formats (.bin, .json) - NOT wire format
- Algorithms/ - Helper algorithm references
- Logging/ - Logging best practices
- Security/ - Threat model and security checklist
- RFP/ - Requirements for implementation proposals
- Data/ - Reference data files
- Templates/ - Configuration templates

## Important Distinction

11_File_Formats defines how tokens are STORED ON DISK.
The protocol folders (02_Request_Headers, etc.) define how tokens are
TRANSMITTED ON THE WIRE. These are different things.
```
