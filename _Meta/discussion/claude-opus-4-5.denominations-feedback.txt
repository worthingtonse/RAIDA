FEEDBACK FOR GPT-4.1 (gpt-4.1.denominations.txt)
=============================================================================

STRENGTHS:
----------
1. Good identification of the core issues (missing names, encoding clarity)
2. Suggests including scientific notation (1e-7) which is helpful for AI
3. Emphasizes the 10^exp formula
4. Good point about break/join rules needing explicit documentation
5. "AI guidance" section is a good idea

ISSUES/SUGGESTIONS:
-------------------
1. NAMING SCHEME: "Micro-1", "Micro-10", "Milli-1", "Milli-10", "Milli-100"
   is systematic but NOT intuitive. These names:
   - Don't follow standard metric prefixes (micro = 10^-6, milli = 10^-3)
   - The numbers (1, 10, 100) add confusion rather than clarity
   - An AI might struggle to map "Micro-10" to 10^-6

2. FORMATTING NOTE: Good catch on ".0000-0001" being non-standard.
   Should use "0.00000001" consistently.

3. MISSING: No proposed names for the positive denominations beyond what
   already exists. Could benefit from shorthand like "Ten-K" for 10,000.


=============================================================================
FEEDBACK FOR GEMINI (gemini.denominations.txt)
=============================================================================

STRENGTHS:
----------
1. EXCELLENT overall structure - leads with the formula, then examples
2. "Fragment" as the base smallest unit is a clean concept
3. Comprehensive table with all columns (Name, Value, ID, Hex, Calculation)
4. Good "Making Change" section explaining break/join
5. Establishes clear terminology: "CloudCoin (CC)" as base unit
6. Two's complement explanation is clear

ISSUES/SUGGESTIONS:
-------------------
1. FRAGMENT NAMING ISSUE: "Giga-Fragment" and "Mega-Fragment" are confusing
   because:
   - "Giga" normally means 10^9 (billion), but here it's 10^-7
   - "Mega" normally means 10^6 (million), but here it's 10^-6
   - This INVERTS the standard metric meaning

   Better alternatives:
   - Use "Deci-Fragment" (10^-7 = 0.1 of 10^-6)
   - Or use completely non-metric names to avoid confusion

2. "10k-Fragment" and "100k-Fragment" use "k" (kilo = 1000) for small
   fractions, which is also counterintuitive.

3. POSITIVE DENOMINATIONS: Names like "100-CloudCoin" are verbose.
   Could use "Hundred" or "C-Note" style shorthand.


=============================================================================
POINTS OF AGREEMENT (ALL 3 MODELS)
=============================================================================

1. The core formula (Value = 10^code) MUST be prominently stated
2. The encoding is signed int8_t, transmitted as uint8_t (two's complement)
3. All fractional denominations need names (current "?" is unacceptable)
4. Break/join rules need explicit documentation
5. Table format should include: Name, Value, Signed Code, Hex Code
6. Scientific notation or clear decimal values should be used
